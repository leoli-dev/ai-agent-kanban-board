# AI Agent Kanban Board(AI 智能体看板)

[English](README.md) | **中文**

自托管的多智能体编码工作流,把一个产品想法变成在本地落地执行的代码项目。你用一段提示词、若干链接和上传的文件创建项目;规划器(planner)智能体会先提澄清问题、再产出计划;你审批通过后,编码器(coder)、评审(reviewer)、测试(tester)、调试(debugger)等智能体便在看板上推进任务。

本应用面向**单个开发者在自己机器上运行智能体**的场景。它把模型凭证保存在本地,把每一次运行实时流式呈现到界面,并在某个模型触发配额或鉴权问题时,自动在多个 AI 服务商之间故障切换。

## 它能做什么

```text
想法 + 文件 + 链接
  -> 规划器智能体 问答(Q&A)
  -> plan.md / plan.json
  -> 人工审批
  -> 看板任务 DAG(有向无环图)
  -> 编码器 / 评审 / 测试 / 调试 智能体
  -> 最终完成报告
```

- 项目仪表盘:展示状态、进度、累计花费,以及后续操作入口。
- 规划器对话:用于澄清问题、审阅计划、请求修改与最终审批。
- 每个项目一个看板:Backlog(待办)、WIP(进行中)、To Review(待评审)、To Test(待测试)、Done(完成),以及 Failed(失败)、Blocked(阻塞)状态。
- 任务详情页:验收标准、实时日志、运行历史、重跑/终止/暂停控制、单任务模型覆盖,以及各阶段小结。
- 把过大或反复失败的任务交还给规划器**拆分为更小的子任务**(以暂停状态创建供你审阅),并就地重接进任务 DAG。
- 单任务有**运行时限(默认 30 分钟)**:超时的任务会被中止并计为一次失败尝试,这样过大的任务会快速失败而不是无限循环。
- 按角色做多服务商模型路由:规划器、任务生成器、编码器、评审、测试、调试,以及编排器测试角色。
- 内置服务商预设:Claude、Codex、DeepSeek、Kimi、Qwen、GLM、MiniMax、OpenRouter、本地 Anthropic 兼容服务,以及基于环境变量的自定义档案。
- 自动配额冷却并回退到下一个已配置模型;鉴权失败会停用该服务商并发出通知。
- 通过隔离的 git worktree 和任务分支并行执行任务,任务完成后自动合并回项目集成分支。
- 活动账本:记录运行时长、状态、模型、token 以及可获取到的花费信息。
- 通知:应用内、macOS 横幅,以及可选的 SMTP 邮件。

## 环境要求

- Node.js 20 或更高版本
- pnpm
- Git
- 一个或多个受支持的智能体 CLI 或 API key:
  - Claude Code CLI(`claude`)用于 `claude-code` 引擎档案
  - OpenAI Codex CLI(`codex`)用于 `codex` 引擎档案
  - Anthropic 兼容的 API 端点,用于 DeepSeek、Kimi、Qwen、GLM、MiniMax、OpenRouter、本地服务或自定义档案
- macOS 为可选项,仅在需要原生横幅通知时才用到。

## 本地运行

```bash
pnpm install
pnpm dev
```

开发模式会启动:

- API 服务:`http://localhost:5713`
- Vite 前端:`http://localhost:5173`

类生产环境的本地运行:

```bash
pnpm build
pnpm start
```

`pnpm start` 会从 `http://<你的IP>:5713` 同时提供构建好的前端和 API,便于在同一局域网的另一台设备上访问。

> 提示:服务端用 `tsx` 直接运行 TypeScript,因此每次 `pnpm start` 都会加载最新的服务端代码。前端改动需要重新 `pnpm build`,因为生产模式下 `:5713` 提供的是预先构建好的 `dist/`(开发时请用 `:5173` 看前端的热更新)。

## 配置

启动应用后,在前端界面打开 Settings(设置)。

1. 在 Models 中添加模型档案。尽量使用引导式预设,或用原始环境变量创建自定义档案。
2. 在 Secrets 中保存 API key。它们以本地文件权限写入 `data/secrets.json`,不会存进数据库。
3. 为每个角色分配有序的模型优先级。每个角色从上到下依次尝试。
4. 调整流水线设置:卡死阈值、运行时限(单任务截止时间,默认 30 分钟)、重试次数、评审打回上限、并行任务数、规划器问答轮数、自动评审、自动测试。
5. 如需 macOS 横幅或 SMTP 邮件通知,请启用通知。

在服务商环境变量中可用 `${SECRET:NAME}` 引用密钥。若在 `data/secrets.json` 中找不到该密钥,服务端会回退到进程环境变量。

常用环境变量:

```bash
AKB_PORT=5713          # API / 生产前端端口
AKB_HOST=0.0.0.0       # 绑定地址
AKB_DATA_DIR=./data    # 数据库、密钥、上传、工作区、日志
```

## 项目工作流

1. 创建项目:填写想法、目标本地仓库路径,可选的参考链接与上传文件。
2. 开始规划。规划器在产出 `plan.md` 和结构化任务前,可能会先提问。
3. 审阅计划。审批通过即开始,或把修改意见退回给规划器。
4. 智能体在专用项目分支上工作,分支名形如 `agent/<project>-<id>`。
5. 相互独立的任务在隔离的 worktree、各自的任务分支上运行。完成的任务会自动合并进项目分支。
6. 评审与测试阶段可自动运行,也可关闭后手动拖动卡片。
7. 如果某个任务过大或反复失败,打开它并使用「拆分为子任务」把它退回规划器;它会就地被更小的子任务替换(以暂停状态创建,供你审阅)。
8. 所有任务完成后,应用会生成完成报告,包含交付物位置、如何运行,以及按任务的明细统计。

你平时的工作区检出与任务 worktree 是相互隔离的。若目标仓库有未提交改动,编排器会拒绝开工,以免覆盖你的本地改动。

## 数据与日志

运行时数据默认位于 `data/` 下:

- `data/app.db` — SQLite 数据库
- `data/secrets.json` — 本地密钥库
- `data/workspaces/<project>/inputs/` — 上传的项目资源
- `data/workspaces/<project>/plan/` — 生成的计划文件
- `data/workspaces/<project>/logs/` — 智能体运行的原始日志
- `data/workspaces/<project>/artifacts/` — 项目产物与报告
- `data/workspaces/<project>/worktrees/` — 集成与任务 worktree

`data/` 目录已被 git 忽略。

## 开发

```bash
pnpm test
pnpm typecheck
pnpm build
```

服务端测试套件使用 `mock` 引擎,因此无需消耗模型 token 即可测试流水线行为。

## 安全须知

- 无头编码智能体以适合单用户开发工具的较宽本地权限运行。
- 请在你掌控的仓库上使用,并在开始项目前提交好重要工作。
- 服务商会消耗真实的 API 额度。测试期间请设置保守的重试、运行时限与并发上限。
- 本应用把密钥保存在本地,不会存入托管的 GitHub 仓库。
